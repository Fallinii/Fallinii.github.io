hexo搭建个人博客

发表于2021-06-01|更新于2021-06-07|配置
|阅读量:

Hexo简介

Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub和Coding上，是搭建博客的首选框架。进入Hexo官网进行详细查看。

1.安装Git

Git下载后有一个Git Bash的命令行工具，用这个工具使用Git。 廖雪峰老师的Git教程
查看Git版本命令 

git --version

2.安装nodejs

nodejs下载选择LTS版本
查看node是否安装成功 

node -v  
npm -v

3.安装hexo

创建一个文件夹blog，在文件夹下直接右键git bash打开
输入命令 

npm install -g hexo-cli

查看hexo版本 

hexo -v

初始化hexo，myblog可以自己取别的名字 

hexo init myblog

进入这个文件夹 

cd myblog  
npm install

安装完成后，指定文件夹目录下有：

  • node_modules：依赖包

  • public：存放生成的页面

  • scaffolds：生成文章的一些模板

  • source：用来存放我的文章

  • themes：主题

  • _config.yml：博客的配置文件


    hexo g
    hexo server

打开hexo的服务，在浏览器输入localhost:4000就能看到生成的博客了。

4.Github创建个人仓库

首先注册一个GitHub账户，在首页new一个repository，创建一个和用户名相同的仓库，Repository name填xxx.github.io,点击create repository完成创建仓库。

5.生成SSH添加到GitHub

回到git bash中,yourname输入GitHub用户名，youremail输入GitHub绑定邮箱

git config --global user.name "yourname"
git config --global user.email "youremail"

检查有没有输入正确

git config user.name
git config user.email

然后创建SSH

ssh-keygen -t rsa -C "youremail"

这时候提示已经生成了.ssh文件夹

ssh就是一个秘钥，文件夹中有两个文件，id_rsa是这台电脑的私人秘钥，id_rsa.pub是公共秘钥，可以随便给别人看。当连接GitHub自己的账户时，它根据公钥匹配我的私钥，能相互匹配时，才能顺利通过git上传你的文件到GitHub上。

在GitHub的setting中，找到SSH and GPG keys的设置选项，点击New SSH key,在指定文件夹中使用命令

cat ~/.ssh/id_rsa.pub

把id_rsa.pub里面的信息复制进去。(ssh-rsa 开头，邮箱结尾)
在gitbash中查看是否成功

ssh -T git@github.com

6.将hexo部署到GitHub

这一步将hexo和GitHub关联起来，将hexo生成的文章部署到GitHub上。
打开站点配置文件_config.yml，翻到最后，YourGithubName填写自己的GitHub账户名

deploy:
  type: git
  repo: https://github.com/YourGithubName/YourGithubName.github.io.git
  branch: master

这时需要先安装部署命令deploy-git，这样才能用命令部署到GitHub

npm install hexo-deployer-git --save

然后

hexo clean
hexo generate
hexo deploy

hexo clean清除了之前生成的东西，可以不加
hexo generate生成静态文章，可以用hexo g缩写
hexo deploy部署文章，可以用hexo d缩写


过一会就能在http://yourname.github.io这个网站看到博客了

7.设置个人域名

  1. 注册阿里云买一个域名，选择自己的域名，进行实名认证。

  2. 在博客文件source中创建一个名为CNAME的文件，不要后缀，把域名写上去。

  3. 在域名控制台中找到自己购买的域名，点击解析，添加解析。

  4. 在自己电脑ping一下网站地址，得到对应的IP地址，添加记录。
    可以使用两种记录类型，CNAME记录值填写域名，A类型记录值填写IP地址

  5. 登录GitHub，进入之前创建的仓库，点击settings，设置Custom domain，输入域名。

  6. 最后在gitbash中输入 

     hexo clean  
 hexo g  
 hexo d

之后打开浏览器，输入自己的域名，就可以看到搭建的网站了。


接下来就可以开始写文章了 

hexo new newblog

然后在source/_post中打开markdown文件，开始编辑文章。写完后再重复一遍

hexo clean  
hexo g  
hexo d

就可以看到更新了。

8.个性化主题

默认的主题是landscape，在官网的主题中可以选择喜欢的主题进行修改。官网的主题
我选的nexT主题


在myblog目录下，进入gitbash，输入

git clone https://github.com/theme-next/hexo-theme-next themes/next

将下载下来的文件放入myblog/themes文件夹里，修改_config.yml文件第100行左右

theme: next

gitbash中更新Hexo

hexo clean  
hexo g  
hexo d

重新打开网站就可以看到主题更新了。

9.其他个性化设置

  1. 基础设置
  2. 添加动态背景
  3. butterfly主题样式美化，很详细

遇到的问题

  1. gitbash要在myblog界面进入
  2. 拿不到id_rsa.pub里的信息，查了命令之后拿到，复制的时候忘了复制前几个字母(ssh-rsa)…
  3. 查看SSH秘钥是否设置成功的时候出现提示 This key is not known by other names / Are you sure you want to continue connecting (yes/no/[fingerprint])?,选择yes。
  4. 购买域名实名认证失败，传来的jpg文件太大，修剪的时候把身份证的边缘剪掉了，没通过，后来改了分辨率调图片大小 就离谱
  5. 域名解析失败，刚开始使用了ping到的IP地址，对应A记录类型，出现错误信息Your site's DNS settings are using a custom subdomain, that's set up as an A record,后来使用CNAME类型，记录值填域名就成功了。
  6. CNAME文件和GitHub的custom domain中都不要加www
